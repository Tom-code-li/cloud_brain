# 模块材料包 — 患者系统（Patient System）

---

## 1. 模块名称 + 负责内容

**模块名称**：患者系统（Patient System）

**负责内容**：本人独立负责该模块的完整后端开发，涵盖以下 8 个子模块的全部 Controller、Service、Mapper、Entity、DTO、Config 层代码及数据库设计：

| 子模块 | 核心内容 |
|:---|:---|
| 患者认证 | 注册、登录、强制登录、退出、改密、患者信息CRUD |
| 挂号预约 | 科室浏览、医生查询、排班查看、预约挂号、取消挂号、余号管理 |
| 费用支付 | 费用列表、待缴账单、在线支付、退费处理 |
| 病历查询 | 病历列表、病历详情 |
| 处方查询 | 处方列表、处方详情、药品明细 |
| 检查检验 | 检查订单查询、检验报告查询、报告详情 |
| AI智能问诊 | 症状提交、AI 智能导诊、风险等级评估 |
| 药品查询 | 药品搜索、药品详情 |

---

## 2. 模块解决的问题（一句话）

为患者提供从**注册登录 → AI导诊 → 挂号预约 → 费用支付 → 病历/处方/检查报告查询**的全流程线上就医服务，解决传统就医过程中挂号难、排队久、信息不透明、不知道挂哪个科室的问题。

---

## 3. 业务流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                         患者全流程就医                                   │
└─────────────────────────────────────────────────────────────────────┘

Step 1 ──── 注册 / 登录
              │
              ▼
Step 2 ──── [AI 智能导诊] 输入症状 → AI 推荐科室 & 评估风险等级
              │
              ▼
Step 3 ──── 挂号预约：浏览科室 → 选择医生 → 查看排班 → 预约 → 支付挂号费
              │
              ▼
Step 4 ──── 就诊后查询：查看病历 / 查看处方及药品明细 / 查看检查检验报告
              │
              ▼
Step 5 ──── 费用管理：查看待缴费用 → 在线支付 → 查看历史费用记录
```

### 详细流程描述

| 步骤 | 业务流程 | 涉及模块 | 对应 API |
|:---:|:---|:---|:---|

| 2 | （可选）患者输入症状描述，AI 导诊助手分析并推荐就诊科室、评估紧急程度 | AI 问诊 | `POST /api/ai/consult` / `POST /api/ai/smart` |

## 6. AI 功能说明

### 6.1 功能概述

系统集成了 **DeepSeek API**，提供**智能导诊分诊**服务。患者描述症状后，AI 自动分析病情、推荐就诊科室并评估风险等级。

### 6.2 技术流程

```
患者输入症状
      │
      ▼
AiConsultationServiceImpl.handleConsultation()
      │
      ├── 1. 查询数据库中所有活跃科室 (DepartmentMapper.findAllActive())
      ├── 2. 调用 AiKnowledgeService.buildCandidates() 构建患者相关候选信息
      ├── 3. 构造 System Prompt（限定角色、输出格式、约束条件）
      ├── 4. 构造 User Prompt（用户模式 + 症状内容 + 候选科室列表 + 候选信息）
      ├── 5. 调用 DeepSeekAiService.chat(systemPrompt, userPrompt)
      ├── 6. 解析 AI 返回的 JSON（AiConsultResponse）
      │       ├── summary        → 病情摘要
      │       ├── riskLevel      → 风险等级（普通 / 紧急）
      │       ├── recommendedDept → 推荐科室
      │       ├── aiResult       → AI 详细分析结果
      │       └── note           → 备注说明
      ├── 7. 科室名称精确匹配 (matchDepartmentId)
      ├── 8. 风险等级归一化 (normalizeRiskLevel)
      └── 9. 保存问诊记录至数据库 (AiConsultation)
```

### 6.3 关键配置

| 配置项 | 值 | 说明 |
|:---|:---|:---|
| API Base URL | `https://api.deepseek.com` | DeepSeek API 地址 |
| Model | `deepseek-v4-flash` | 使用的模型 |
| API Key | `${DEEPSEEK_API_KEY}` | 环境变量注入，默认 `CHANGE_ME` |
| 调用方式 | `WebClient` (Spring WebFlux) | 异步 HTTP 客户端 |

### 6.4 Prompt 设计要点

- **System Prompt**：限定 AI 扮演医院导诊助手，只能基于给定候选项回答，不得编造数据库不存在的科室/医生/信息；要求输出严格 JSON 格式。
- **User Prompt**：包含用户模式、用户输入内容、可选科室列表（名称+简介）和患者相关候选项（病历/检查/处方等），约束 AI 只从候选项中选择。

### 6.5 容错处理

- JSON 解析失败时使用默认值兜底（风险=普通，科室=空）
- 多关键词匹配风险等级（支持中文"急""危"及英文 urgent/critical 等）
- 科室名称精确匹配，匹配不到时返回 null

## 8. 难点与解决方案

### 难点 1：挂号并发控制与余号一致性

**问题描述**：
在高峰期大量患者同时预约同一医生时，余号扣减可能出现超卖（余号变为负数）或重复预约同一时段的问题。`RegistrationServiceImpl.registerAppointment()` 涉及三步操作：① 查排班余号 → ② 插入挂号记录 → ③ 扣减余号，非原子操作，多线程并发下存在数据竞争。

**解决方案**：
- **数据库层面**：在 `doctor_schedule` 表的 `remain_quota` 字段上使用 **乐观锁机制**（MyBatis-Plus 的 `@Version` 注解或 SQL 条件更新 `UPDATE doctor_schedule SET remain_quota = remain_quota - 1 WHERE schedule_id = ? AND remain_quota > 0`），确保扣减操作是原子的。
- **事务层面**：使用 `@Transactional` 声明式事务包裹整个预约流程，保证 `INSERT registration` 与 `UPDATE doctor_schedule` 在同一数据库事务中，任一失败则整体回滚。
- **业务校验**：挂号前二次检查排班状态（`status` 是否为"可预约"、`remain_quota > 0`），不满足时直接抛出 `IllegalStateException`。
- **取消恢复**：取消挂号时同样在事务内恢复 `remain_quota + 1`，保证余号守恒。

### 难点 2：AI 导诊结果的结构化解析与稳定性

**问题描述**：
DeepSeek API 返回的是自然语言文本，期望解析为结构化 JSON（含 `summary`、`riskLevel`、`recommendedDept`、`aiResult`、`note` 字段）。但 LLM 存在输出格式不稳定（缺少字段、字段名不一致、嵌套额外内容）的问题，直接解析失败会导致整个导诊流程崩溃。

**解决方案**：
- **多层容错解析**：使用 `ObjectMapper.readValue()` 尝试解析 JSON，捕获所有异常（`Exception ignored`），解析失败时使用全量默认值兜底，绝不抛出异常。
- **Prompt 强约束**：System Prompt 中明确要求"输出必须是 JSON"，User Prompt 中再次强调"请只从候选项中选择，并输出严格 JSON"，双重约束降低格式漂移概率。
- **字段归一化处理**：
  - `riskLevel` 归一化：支持中文"紧急""危"及英文 `urgent`/`critical`/`high`/`warning` 等多关键词匹配，统一映射为系统内部枚举值。
  - `recommendedDept` 精确匹配：使用数据库科室全称流式匹配，不采用模糊匹配以避免歧义。
- **回退策略**：若 AI 匹配不到合适科室，`recommendedDeptId` 置为 null，`note` 中说明未找到匹配项，前端可展示 AI 原始文本供患者参考。
